## What this chunk does

Splits parsed elements into embeddable chunks.
Each chunk becomes a vector in Qdrant.
The quality of chunking directly affects retrieval quality.

## Why chunking matters

- LLM context windows are limited (8K-128K tokens)
- Embedding models work best with 200-1000 token inputs
- Relevant context must be grouped together
- Tables must NOT be split (they break)
- Metadata enables filtering ("only search structural documents")

## Chunk Model

```python
@dataclass
class Chunk:
    id: str                    # UUID
    content: str               # Text to embed
    metadata: dict             # project_id, document_id, section, page, etc.
    embedding_id: str | None   # Qdrant point ID (set after embedding)
```

## Chunking Strategies

### Semantic Chunking (preferred)

Uses LangChain's SemanticChunker with BGE embeddings.
Splits text where semantic meaning changes between sentences.

```
Sentence 1: "M25 concrete is used for foundations." (topic: concrete)
Sentence 2: "The foundation depth should be 1.5m." (topic: foundation depth)
Sentence 3: "Steel reinforcement Fe500 is required." (topic: steel)

Similarity(1,2) = 0.7 (related: foundation + concrete)
Similarity(2,3) = 0.3 (different: foundation vs steel)

Split between 2 and 3 because similarity drops below threshold.
```

### Fixed-Size Chunking (fallback)

Used when SemanticChunker is not available or text is short.
Splits at sentence boundaries to avoid breaking mid-thought.

```
Text: 5000 chars
Chunk size: 1000 chars
Overlap: 200 chars

Chunk 1: chars 0-1000
Chunk 2: chars 800-1800 (200 char overlap)
Chunk 3: chars 1600-2600
...
```

## Chunking Flow

```
Parsed Elements
  │
  ├─→ Text elements (NARRATIVE, LIST, UNKNOWN)
  │     → Accumulate into text buffer
  │     → When buffer > chunk_size:
  │         → Semantic split (or fixed split)
  │         → Create Chunks with metadata
  │         → Overlap between chunks
  │
  ├─→ Table elements
  │     → Convert to markdown: | col1 | col2 |\n|---|---|\n| val1 | val2 |
  │     → Keep as SINGLE chunk (never split tables)
  │     → If table too large: split by rows, repeat header in each
  │
  ├─→ Image elements
  │     → Create chunk: "[Image: floor_plan.png]\nPath: /tmp/img.png"
  │     → Store image_path in metadata for later vision analysis
  │
  └─→ Title elements
        → Track as current_section
        → Include section name in metadata of following chunks
```

## Metadata Extraction

### TextMetadata Model

```python
@dataclass
class TextMetadata:
    char_count: int              # Total characters
    word_count: int              # Total words
    has_numbers: bool            # Contains numeric values
    has_measurements: bool       # Contains "300mm", "4.5m", "M25"
    measurements: list[str]      # Extracted: ["M25", "Fe500", "300mm"]
    has_code_references: bool    # Contains "IS 456", "NBC 2016"
    code_references: list[str]   # Extracted: ["IS 456", "IS 875"]
    language: str                # "en" or "non-english"
    keywords: set[str]           # Technical keywords found
    is_technical: bool           # Contains construction terminology
```

### Measurement Detection Patterns

```python
MEASUREMENT_PATTERNS = [
    r"\d+\.?\d*\s*(?:mm|cm|m|km|kg|g|mg|l|ml|sq\.?\s*m|cum)",  # Units
    r"\d+\.?\d*\s*(?:x|×)\s*\d+\.?\d*\s*(?:mm|cm|m)",           # Dimensions
    r"M\d+",      # M20, M25, M30 (concrete grades)
    r"Fe\d+",     # Fe415, Fe500 (steel grades)
    r"\d+\.?\d*\s*(?:MPa|GPa|N/mm2)",                            # Stress units
]
```

### Code Reference Detection

```python
CODE_PATTERNS = [
    r"IS\s*\d+",   # IS 456, IS 875
    r"NBC\s*\d*",  # NBC 2016
    r"CPWD",
    r"SECTION\s*\d+",
    r"CLAUSE\s*\d+",
]
```

## Usage

```python
# Parse
parser = DocumentParser()
elements = await parser.parse("samples/boq.pdf")

# Chunk
chunker = Chunker(chunk_size=1000, chunk_overlap=200, use_semantic=True)
chunks = await chunker.chunk(elements, document_id="doc-1", project_id="proj-1")

# Result
for chunk in chunks:
    print(f"{chunk.id[:8]}... | {len(chunk.content)} chars | {chunk.metadata.get('section')}")
```

## Edge Cases

| Case | Handling |
|------|----------|
| Empty elements | Skipped (no chunk created) |
| Very short text (< 50 chars) | Merged with next chunk |
| Very large table | Split by rows, header repeated |
| Mixed languages | Detected and tagged |
| Special characters | Preserved in content |
| Unicode (CJK, Arabic) | Handled by chunker |

## Verification

```python
chunker = Chunker(chunk_size=500, chunk_overlap=100)

elements = [
    ParsedElement(type=ElementType.NARRATIVE, content="A" * 2000),
    ParsedElement(type=ElementType.TABLE, content="", table_data=[["A", "B"], ["1", "2"]]),
]

chunks = await chunker.chunk(elements, document_id="doc-1", project_id="proj-1")

# Text chunked into pieces
text_chunks = [c for c in chunks if c.metadata.get("chunk_type") != "table"]
assert len(text_chunks) >= 2

# Table is single chunk
table_chunks = [c for c in chunks if c.metadata.get("chunk_type") == "table"]
assert len(table_chunks) == 1
```
