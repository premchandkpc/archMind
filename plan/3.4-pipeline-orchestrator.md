## What this chunk does

Coordinates the full ingestion pipeline via Redis Streams.
Upload → Parse → Chunk → Embed → Graph → Index → Ready.
Each step is independent, retryable, and horizontally scalable.

## Why Event-Driven

| Synchronous | Event-Driven |
|-------------|--------------|
| API blocks for 5min on 100-page PDF | API returns 202 immediately |
| One failure kills entire pipeline | Each step retried independently |
| Scale = bigger machine | Scale = more workers |
| No backpressure | Redis XREADGROUP handles backpressure |

### civilmind/events/workers.py

**Purpose:** Background worker with consumer group.

```python
class PipelineWorker:
    def __init__(self, stream: str, group: str, handler: Callable):
        self.stream = stream
        self.group = group
        self.handler = handler
    
    async def run(self):
        while True:
            event = await self.consumer.consume(self.stream, self.group)
            if event:
                try:
                    await self.handler(event)
                    await self.consumer.ack(event.id)
                except RetryableError:
                    await self.consumer.nack(event.id)
                except PermanentError:
                    await self.consumer.dead_letter(event.id)
```

## Retry Strategy

| Attempt | Delay | Action |
|---------|-------|--------|
| 1 | 0s | Try |
| 2 | 5s | Retry with backoff |
| 3 | 25s | Final retry |
| 4+ | — | Dead-letter queue |

## Failure Scenarios

| Scenario | Behavior |
|----------|----------|
| Parser fails | Event nacked, retried with backoff |
| Worker crashes | Unacknowledged events redelivered |
| Redis down | Pipeline pauses, queries still work |
| Embedding API rate limited | Retry with exponential backoff |
| Neo4j down | Graph step skipped, document marked "partial" |

## Observability

- Counter: events_published_total (by stream, event_type)
- Counter: events_processed_total (by stream, handler, status)
- Histogram: handler_duration_seconds (by handler)
- Gauge: queue_depth (by stream)

## Security

- Worker runs as separate Docker container (blast radius)
- Dead-letter queue monitored for persistent failures
- Event payload size limit (10MB)
