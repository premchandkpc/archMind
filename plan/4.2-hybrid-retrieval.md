## What this chunk does

The most important chunk in the RAG pipeline.
Combines BM25 keyword search + vector semantic search.
Merges with Reciprocal Rank Fusion.
Reranks with cross-encoder.
Compresses context to remove noise.

## Why hybrid over pure vector

| Scenario | Vector only | BM25 only | Hybrid |
|----------|-------------|-----------|--------|
| "IS456 clause 5.3" | Misses exact code | Finds exact | Finds exact |
| "concrete grade" | Finds semantic matches | Misses synonyms | Finds both |
| "M25 for foundation" | Good | Good | Best |
| Technical jargon | Medium | Good | Best |

Hybrid combines the strengths of both approaches.

## Retrieval Pipeline

```
User Query: "What is the concrete grade for foundations?"

Step 1: BM25 Search (top 20)
  → Tokenize query: ["concrete", "grade", "foundations"]
  → Search BM25 index
  → Returns: chunks with keyword matches

Step 2: Vector Search (top 20)
  → Embed query: [0.1, 0.3, ..., 0.7] (768 dims)
  → Search Qdrant with cosine similarity
  → Returns: chunks with semantic similarity

Step 3: RRF Merge (top 30)
  → Combine rankings from both searches
  → Reciprocal Rank Fusion: score = sum(1/(k + rank))
  → k=60 (standard from original paper)

Step 4: Cross-Encoder Rerank (top 10)
  → For each (query, chunk) pair, compute relevance score
  → Cross-encoder is more accurate but slower
  → Sort by reranked score

Step 5: Context Compression (top 5)
  → Remove irrelevant sentences from each chunk
  → Keep sentences with highest query overlap
  → Reduce token count for LLM
```

## BM25Index

### How BM25 works

BM25 scores documents based on:
- **Term frequency (TF):** How often the query term appears in the document
- **Inverse document frequency (IDF):** How rare the term is across all documents
- **Document length normalization:** Penalizes very long documents

Formula: `score = sum(IDF(t) * TF(t, D) * (k1 + 1)) / (TF(t, D) + k1 * (1 - b + b * |D|/avgdl))`

### Tokenization

```python
def _tokenize(text: str) -> list[str]:
    text = text.lower()
    text = re.sub(r"[^a-z0-9\s]", " ", text)
    return [t for t in text.split() if len(t) > 1]
```

## CrossEncoderReranker

### How reranking works

1. Create (query, chunk) pairs
2. Feed pairs to cross-encoder model
3. Cross-encoder computes relevance score for each pair
4. Sort by score (higher = more relevant)
5. Return top_k

**Why cross-encoder is better than bi-encoder:**
- Bi-encoder (embedding search) encodes query and document separately
- Cross-encoder sees both query and document together
- Cross-encoder can attend to interactions between query and document
- More accurate but slower (can't pre-compute)

### Model

```
cross-encoder/ms-marco-MiniLM-L-6-v2
- Trained on MS MARCO passage ranking
- 6-layer MiniLM
- Fast inference (~1ms per pair)
```

## ContextCompressor

### Extractive compression

Keeps only the most relevant sentences from each chunk:
1. Split chunk into sentences
2. Score each sentence by query word overlap
3. Boost early sentences (position bias)
4. Keep top N sentences (at least 2, at most half)

This reduces noise and focuses the LLM on relevant content.

## RRF Algorithm

```python
def _rrf_merge(vector_results, bm25_results, k=60):
    scores = {}
    
    for rank, doc in enumerate(vector_results):
        if doc.id not in scores:
            scores[doc.id] = {"doc": doc, "score": 0}
        scores[doc.id]["score"] += 1 / (k + rank + 1)
    
    for rank, doc in enumerate(bm25_results):
        if doc.id not in scores:
            scores[doc.id] = {"doc": doc, "score": 0}
        scores[doc.id]["score"] += 1 / (k + rank + 1)
    
    merged = sorted(scores.values(), key=lambda x: x["score"], reverse=True)
    return [item["doc"] for item in merged]
```

**Why k=60:**
- Original RRF paper uses k=60
- Balances influence of high-ranking vs low-ranking documents
- Higher k = more uniform distribution (less weight on top ranks)
- Lower k = more weight on top ranks

## Usage

```python
retriever = HybridRetriever(
    qdrant=store,
    bm25_index=bm25,
    embedder=embedder,
    reranker=CrossEncoderReranker(),
    compressor=ContextCompressor(),

results = await retriever.retrieve(
    query="What is the concrete grade for foundations?",
    project_id="proj-1",

for r in results:
    print(f"[{r.score:.3f}] {r.source}: {r.content[:80]}...")
```

## Evaluation

To measure retrieval quality:
1. **Recall@5** — are relevant docs in top 5?
2. **MRR** — mean reciprocal rank of first relevant result
3. **NDCG** — normalized discounted cumulative gain
4. **Latency** — total retrieval time should be < 500ms
