# 11.4 — CAD API + Background Worker

**Status:** Planned

---

## What this chunk does

Exposes the CAD conversion pipeline via REST API with async job management:
- `POST /cad/convert` — start conversion, returns 202 with job_id
- `GET /cad/status/{job_id}` — poll job progress
- `GET /cad/download/{job_id}.{ext}` — download the output file
- Background `CadWorker` listening on Redis Stream `cad:conversion`
- Job status stored in PostgreSQL

## Dependency

Requires chunks 11.1-11.3 (full conversion pipeline). This chunk exposes it over the network.

## Pipeline

```
POST /cad/convert
  → validate file_id exists on MinIO
  → create job record (status=queued)
  → publish cad:conversion event to Redis Stream
  → return 202 { job_id, status_url }

CadWorker (background asyncio task, started in app lifespan):
  → BLREAD cad:conversion stream
  → for each job:
      → update status to "processing"
      → download file from MinIO to temp dir
      → run CadConverter.convert()
      → upload output file to MinIO
      → update status to "done" with download_url
      → on error: update status to "error" with error message
  → handle PermanentError / RetryableError (same pattern as PipelineWorker)

GET /cad/status/{job_id}
  → query job from PostgreSQL
  → return { status, progress, message, download_url }

GET /cad/download/{job_id}.{ext}
  → redirect to presigned MinIO URL (or stream)
```

### civilmind/events/cad_handlers.py

**Purpose:** Handle `cad:conversion` events dispatched by the worker.

```python
from __future__ import annotations

import structlog

from civilmind.cad.converter import CadConverter
from civilmind.events.errors import PermanentError, RetryableError
from civilmind.storage.minio_storage import MinIOStorage

logger = structlog.get_logger()

async def handle_cad_conversion(event: dict, storage: MinIOStorage) -> dict:
    """Handle cad:conversion event → run converter → store output."""
    file_id = event.get("file_id")
    output_format = event.get("output_format", "dxf")
    job_id = event.get("job_id")

    try:
        # 1. Download file from MinIO
        local_path = await storage.download(file_id)

        # 2. Run conversion
        converter = CadConverter()
        result = await converter.convert(
            file_path=local_path,
            output_format=output_format,
            on_progress=lambda p, msg: _update_progress(job_id, p, msg),
        )

        if not result.success:
            raise PermanentError(result.error or "Conversion failed")

        # 3. Upload output to MinIO
        ext = output_format
        output_key = f"cad-output/{job_id}.{ext}"
        await storage.upload(result.output_path, output_key)

        return {
            "status": "done",
            "download_url": f"/api/v1/cad/download/{job_id}.{ext}",
            "file_size_bytes": os.path.getsize(result.output_path),
            "element_count": result.element_count,
        }

    except Exception as e:
        logger.error("CAD conversion failed", job_id=job_id, error=str(e))
        raise
```

## Wiring

### Register router

In `civilmind/api/app.py`:
```python
from civilmind.api.routes.cad import router as cad_router
app.include_router(cad_router, prefix="/api/v1")
```

### Start CadWorker

In `civilmind/api/app.py` lifespan:
```python
async def lifespan(app):
    event_bus = EventBus()
    await event_bus.start()
    # Start existing workers
    pipeline_worker = PipelineWorker(...)
    # Add:
    cad_worker = CadWorker(storage, db_session_factory)
    workers = [asyncio.create_task(pipeline_worker.run()),
               asyncio.create_task(cad_worker.run())]
    yield
    for w in workers:
        w.cancel()
    await event_bus.stop()
```

## Limitations

- No WebSocket progress push — client must poll
- No concurrent job limiting — each conversion runs as a separate task
- No job cancellation — once started, a conversion runs to completion
