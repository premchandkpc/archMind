## What this chunk does

Converts text chunks into vector embeddings for similarity search.
Embeddings capture semantic meaning: "concrete grade" and "cement mix" produce similar vectors.

## Why embeddings matter

- Keyword search (BM25) finds exact matches
- Vector search finds semantic similarity
- Embeddings enable "find documents about X" even if X is described differently
- Quality of embeddings directly affects retrieval quality

## Files to create

### civilmind/pipeline/embedder.py

**Purpose:** Embedding service with BGE/OpenAI backends and caching.

## Embedding Providers

### BGE-base (Primary)

| Property | Value |
|----------|-------|
| Model | BAAI/bge-base-en-v1.5 |
| Dimension | 768 |
| Cost | Free |
| Speed | ~50 texts/sec (local) |
| Quality | Good |
| Offline | Yes |
| Rate limit | None |

Best for: development, testing, high-volume processing.

### OpenCode Zen (Fallback)

| Property | Value |
|----------|-------|
| Model | Via OpenCode Zen API |
| Dimension | 1536 |
| Cost | Pay-per-token |
| Speed | ~100 texts/sec (API) |
| Quality | Best |
| Offline | No |
| Rate limit | Provider-dependent |

Best for: production, highest quality retrieval.

## Class Hierarchy

```python
  └── CachedEmbedder       # Wrapper adding disk cache
```

## EmbedderFactory

```python
embedder = EmbedderFactory.create(provider="bge")  # or "opencode"
```

## CachedEmbedder

Wraps any embedder with disk-based caching:

```
Text "M25 concrete grade"
  → SHA256 hash: "a1b2c3d4..."
  → Check .cache/embeddings/a1b2c3d4.npy
  → If exists: load and return (0ms)
  → If not: compute embedding, save to cache, return
```

This avoids re-embedding the same text across runs.

## Embedding Flow

```
Text chunk
  → CachedEmbedder.embed(text)
    → Compute SHA256 hash
    → Check cache directory
    → If cached: load numpy array, convert to list
    → If not cached:
        BGE: model.encode(text, normalize=True) → 768-dim list
        OpenCode Zen: client.embeddings.create(text) → 1536-dim list
    → Save to cache as .npy file
    → Return list[float]
```

## Batch Embedding

```python
# Embed 100 chunks at once
vectors = await embedder.embed_batch([chunk.content for chunk in chunks])
# Returns: list of 100 vectors, each 768 or 1536 dimensions

# OpenCode Zen batches in groups of 2048 (API limit)
# BGE uses sentence-transformers batch encoding (32 at a time)
```

## Dimension Matching

When switching between BGE (768) and OpenCode Zen (1536):
- Qdrant collection must match the embedding dimension
- Cannot mix dimensions in same collection
- Recommendation: use one provider consistently

## Usage

```python
from civilmind.pipeline.embedder import EmbedderFactory, CachedEmbedder

# Create embedder
embedder = EmbedderFactory.create(provider="bge")

# Single embed
vector = await embedder.embed("What is the concrete grade for M25?")
assert len(vector) == 768

# Batch embed
vectors = await embedder.embed_batch(["text1", "text2", "text3"])
assert len(vectors) == 3
assert all(len(v) == 768 for v in vectors)

# Deterministic (same input = same output)
v1 = await embedder.embed("hello world")
v2 = await embedder.embed("hello world")
assert v1 == v2

# With caching
cached = CachedEmbedder(embedder)
v1 = await cached.embed("hello")  # computes + caches
v2 = await cached.embed("hello")  # loads from cache (instant)
assert v1 == v2
```
