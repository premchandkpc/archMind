## What this chunk does

Redis Streams for event-driven ingestion pipeline.
Publish/subscribe pattern with consumer groups for parallel processing.

## Why Redis Streams

| Feature | Redis Streams | Kafka | RabbitMQ |
|---------|--------------|-------|----------|
| Already in stack | Yes (cache) | No | No |
| Consumer groups | Yes | Yes | Yes |
| Backpressure | XREADGROUP | Yes | Yes |
| Ops complexity | Low | High | Medium |
| Throughput | ~100k msgs/sec | ~1M msgs/sec | ~50k msgs/sec |

class EventConsumer:

## Failure scenarios

| Scenario | Behavior |
|----------|----------|
| Redis down | Pipeline pauses, queries still work |
| Consumer crash | Unacknowledged events redelivered |
| Event too large | Reject with error (10MB limit) |
