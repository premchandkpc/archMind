# 11.1 — Drawing Recognition: Lines + Symbols + Text

**Status:** Planned

---

## What this chunk does

Rasterizes a drawing (PDF/PNG/JPG) and detects the low-level primitives:
- Lines (Canny edge + Hough transform)
- Circles/arcs (Hough circles)
- Architectural symbols (template matching)
- Text regions (OCR via PaddleOCR)

Output is raw detected primitives — no geometry classification yet.

## Why split here

Detection is the foundation. Building geometry and exporting to CAD/BIM are downstream. This chunk is independently testable (drawing in → JSON of detected primitives out).

## Conversion pipeline (chunk scope)

```
Input: structural_plan.pdf
  1. Rasterize → PNG (300 DPI)
  2. Grayscale + Gaussian blur
  3. Canny edge detection
  4. Hough line + circle detection
  5. Merge collinear segments
  6. Template match for symbols
  7. OCR for text regions
Output: DetectedPrimitives(lines, arcs, circles, symbols, texts)
```

### civilmind/cad/detector.py

**Purpose:** Detect geometric primitives from rasterized drawings.

```python
class LineDetector:
    """Detect lines, circles, arcs from a rasterized drawing."""

    def detect_lines(self, image: np.ndarray) -> list[Line]:
        """Canny edge → Hough lines → merge collinear → return."""

    def detect_circles(self, image: np.ndarray) -> list[Circle]:
        """Hough circle detection."""

    def detect(self, image: np.ndarray) -> DetectedPrimitives:
        """Run all detections and return combined result."""
```

Parameters (config constants):
```python
CANNY_LOW = 50
CANNY_HIGH = 150
GAUSSIAN_KERNEL = (5, 5)
HOUGH_RHO = 1.0
HOUGH_THETA = np.pi / 180
HOUGH_THRESHOLD = 100
HOUGH_MIN_LINE_LENGTH = 30
HOUGH_MAX_LINE_GAP = 10
COLLINEAR_ANGLE_TOLERANCE = 2   # degrees
COLLINEAR_DISTANCE_TOLERANCE = 5  # pixels
```

Algorithm:
1. Convert image to grayscale
2. Apply Gaussian blur
3. Canny edge detection
4. Probabilistic Hough line transform
5. Merge near-collinear segments (angle diff < 2°, endpoint distance < 5px)
6. Hough circles (optional, controlled by `detect_circles` flag)
7. Return `Line[]` + `Circle[]`

### civilmind/cad/symbol_recognizer.py

**Purpose:** Recognize architectural symbols via multi-scale template matching.

```python
class SymbolRecognizer:
    """Match door, window, column, grid marker symbols using templates."""

    TEMPLATES_DIR = Path(__file__).parent / "templates"

    def __init__(self):
        self._templates: dict[str, np.ndarray] = {}
        self._load_templates()

    def _load_templates(self):
        """Load PNG templates from templates/ directory."""

    def detect_symbols(self, image: np.ndarray) -> list[Symbol]:
        """Multi-scale rotated template matching."""
```

Matching logic:
```python
for name, template in self._templates.items():
    for scale in [0.8, 1.0, 1.2]:
        for angle in [0, 90, 180, 270]:
            rotated = rotate_and_scale(template, angle, scale)
            result = cv2.matchTemplate(image, rotated, cv2.TM_CCOEFF_NORMED)
            matches = np.where(result >= 0.7)
            for pt in zip(*matches[::-1]):
                symbols.append(Symbol(name, pt, angle, result[pt[1], pt[0]]))
```

## Limitations

- Templates are fixed-size; scale variation beyond 0.8-1.2 misses
- No angle variation beyond 0/90/180/270 — diagonal symbols missed
- Hough circles struggle with partial arcs
- Requires clean, high-contrast drawings — no hand-drawn sketches
