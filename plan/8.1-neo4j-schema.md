## What this chunk does

Defines the graph schema (12 node types, 12 relationship types).
Ingests entities from parsed documents and floor plan analysis.
Creates the knowledge graph that enables multi-hop reasoning.

## Why knowledge graphs

- Enable multi-hop queries: "What vendor supplies concrete for Bedroom 1?"
- Show relationships between entities
- Enable graph traversal (Building → Floor → Room → Wall → Material → Vendor)
- Complement vector search with structured relationships

## Graph Schema

### Node Types

| Node | Properties | Example |
|------|------------|---------|
| Building | id, name, area_sqm | "Block A", 180.0 |
| Floor | id, level, name | 1, "Ground Floor" |
| Room | id, name, area_sqm, type | "Bedroom 1", 14.5 |
| Wall | id, type, thickness_mm | "load_bearing", 230 |
| Column | id, size_mm, material | "300x300", "RCC" |
| Beam | id, size_mm, span_m | "230x450", 4.0 |
| Door | id, type, width_mm | "swing", 900 |
| Window | id, type, width_mm | "casement", 1200 |
| Material | id, name, grade | "Concrete", "M25" |
| Vendor | id, name, location | "ACC Cement", "Hyderabad" |
| BuildingCode | id, name, section | "IS 456", "5.3" |
| Project | id, name, location | "Apartment Complex", "Hyderabad" |

### Relationship Types

| From | Relationship | To | Example |
|------|--------------|----|---------|
| Building | HAS_FLOOR | Floor | Block A → Ground Floor |
| Floor | HAS_ROOM | Floor | Ground Floor → Bedroom 1 |
| Room | HAS_WALL | Wall | Bedroom 1 → Wall W1 |
| Wall | HAS_DOOR | Door | Wall W1 → Door D1 |
| Wall | HAS_WINDOW | Window | Wall W1 → Window W1 |
| Floor | HAS_COLUMN | Column | Ground Floor → Column C1 |
| Column | SUPPORTS | Beam | Column C1 → Beam B1 |
| Beam | USES_MATERIAL | Material | Beam B1 → Concrete M25 |
| Wall | USES_MATERIAL | Material | Wall W1 → Brick |
| Material | SUPPLIED_BY | Vendor | Concrete → ACC Cement |
| Building | FOLLOWS_CODE | BuildingCode | Block A → IS 456 |
| Project | HAS_BUILDING | Building | Apartment Complex → Block A |

## Neo4jStore API

```python
class Neo4jStore:
    def __init__(self, uri, user, password)
    async def close()
    async def create_constraints()
    async def create_entity(entity) -> str
    async def create_relationship(rel)
    async def query(cypher, params) -> list[dict]
    async def get_entity(label, entity_id) -> dict | None
    async def traverse(start_id, relationship, max_depth) -> list[dict]
    async def delete_project(project_id)
```

## Usage

```python
store = Neo4jStore(uri="bolt://localhost:7687", user="neo4j", password="password")

await store.create_constraints()

# Create entities
building = GraphEntity(label="Building", properties={"id": "B1", "name": "Block A"})
await store.create_entity(building)

# Create relationship
rel = GraphRelationship(from_id="B1", to_id="F1", rel_type="HAS_FLOOR")
await store.create_relationship(rel)

# Traverse
result = await store.traverse("B1", "HAS_FLOOR", max_depth=2)
```
