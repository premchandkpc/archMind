## What this chunk does

Defines the TypedDict that flows through the LangGraph workflow.
Every node reads from and writes to this state.
LangGraph handles serialization and checkpointing.

## Why TypedDict over Pydantic

- LangGraph requires TypedDict for state
- TypedDict supports Annotated reducers (add_messages)
- Pydantic models don't work with LangGraph state

## State Fields

```python
class ProjectState(TypedDict):
    # Input
    project_id: str
    question: str
    document_ids: list[str]
    
    # Retrieval
    retrieved_chunks: Annotated[list[RetrievedChunk], add_messages]
    
    # Analysis results
    drawing_analysis: DrawingAnalysis | None
    violations: Annotated[list[Violation], add_messages]
    cost_estimation: CostEstimate | None
    schedule: Schedule | None
    risk_assessment: RiskAssessment | None
    
    # Review
    review_feedback: ReviewFeedback | None
    
    # Output
    final_answer: str | None
    
    # Workflow control
    messages: Annotated[list[dict], add_messages]
    iteration: int
    needs_human_approval: bool
    current_node: str
    next_nodes: list[str]
    
    # Memory
    context: dict
```

## Annotated Reducers

```python
retrieved_chunks: Annotated[list[RetrievedChunk], add_messages]
```

**Why Annotated:**
- LangGraph uses reducers to combine state from multiple nodes
- `add_messages` appends new items to existing list
- Without Annotated, new state overwrites old state
- With Annotated, retrieved_chunks from node A + node B are merged

## State Factory

```python
def create_initial_state(
    project_id: str,
    question: str,
    document_ids: list[str] | None = None,
) -> ProjectState:
    return ProjectState(
        project_id=project_id,
        question=question,
        document_ids=document_ids or [],
        retrieved_chunks=[],
        drawing_analysis=None,
        violations=[],
        cost_estimation=None,
        schedule=None,
        risk_assessment=None,
        review_feedback=None,
        final_answer=None,
        messages=[],
        iteration=0,
        needs_human_approval=False,
        current_node="start",
        next_nodes=[],
        context={},
    )
```

## State Transitions

```
START
  project_id, question, document_ids set
  iteration=0, messages=[]

AFTER PLANNER
  messages += [{"role": "planner", "content": "Tasks: ..."}]
  next_nodes = ["retriever", "estimator"]

AFTER RETRIEVER
  retrieved_chunks += [chunk1, chunk2, ...]
  context["retrieval_summary"] = "Found 15 chunks"

AFTER ESTIMATOR
  cost_estimation = {"total": 450000, ...}
  context["cost_summary"] = "Estimated 450k"

AFTER REVIEWER
  if is_valid:
    next_nodes = ["reporter"]
  else:
    next_nodes = ["planner"]  # loop
    iteration += 1

AFTER REPORTER
  final_answer = "..."
  -> END
```
