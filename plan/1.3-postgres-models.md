## What this chunk does

Defines the relational database schema using SQLAlchemy async ORM.
Creates 4 tables that track the entire lifecycle of projects, documents, chunks, and entities.
Sets up Alembic for schema versioning and migrations.

## Why PostgreSQL

| Feature | PostgreSQL | SQLite | MySQL |
|---------|------------|--------|-------|
| JSON support | JSONB (indexed) | Limited | JSON |
| Async support | asyncpg | aiosqlite | aiomysql |
| Full-text search | ts_vector | FTS5 | Full-text |
| ACID | Yes | Yes | Yes |
| Scalability | Excellent | Single file | Good |
| Extensions | PostGIS, pg_trgm | None | Limited |

PostgreSQL is the industry standard for production applications.

## Why SQLAlchemy async

- Type-safe ORM (catches errors at development time)
- Async operations (non-blocking I/O)
- Relationship management (cascade deletes)
- Migration support (Alembic integration)

## Files to create

### civilmind/db/engine.py

**Purpose:** Async SQLAlchemy engine with connection pooling.

**What it does:**
1. Creates async engine pointing to PostgreSQL
2. Configures connection pool (size=20, overflow=10)
3. Creates session factory for dependency injection
4. Defines Base class for all ORM models

**Connection pool settings:**
| Setting | Value | Why |
|---------|-------|-----|
| pool_size | 20 | Keep 20 connections ready (avoids connection overhead) |
| max_overflow | 10 | Allow 10 extra under load (handles traffic spikes) |
| pool_pre_ping | True | Test connections before use (handles stale connections) |
| echo | DEBUG only | Log SQL in debug mode (no overhead in prod) |

**Session factory:**
```python
async_session_factory = async_sessionmaker(
    engine,
    class_=AsyncSession,
    expire_on_commit=False,  # Don't reload after commit
)
```

### civilmind/db/models.py

**Purpose:** 4 SQLAlchemy ORM models representing the data schema.

**Why these 4 models:**
- `Project` — top-level container (every query belongs to a project)
- `Document` — uploaded file metadata (tracks version, OCR status)
- `Chunk` — embeddable text pieces (links to Qdrant via embedding_id)
- `Entity` — knowledge graph nodes (links to Neo4j via neo4j_id)

**Model: Project**
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | VARCHAR(36) | PK | UUID primary key |
| name | VARCHAR(255) | NOT NULL | Project name |
| description | TEXT | NULLABLE | Optional description |
| status | VARCHAR(50) | DEFAULT 'draft' | draft → processing → completed/failed |
| created_at | TIMESTAMP | DEFAULT NOW() | Creation time |
| updated_at | TIMESTAMP | DEFAULT NOW() | Last modification time |

**Model: Document**
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | VARCHAR(36) | PK | UUID primary key |
| project_id | VARCHAR(36) | FK → projects.id, CASCADE | Parent project |
| filename | VARCHAR(500) | NOT NULL | Original filename (e.g., "floor_plan.pdf") |
| file_type | VARCHAR(10) | NOT NULL | Extension (e.g., ".pdf") |
| file_size | INTEGER | NOT NULL | Size in bytes |
| storage_path | VARCHAR(1000) | NOT NULL | MinIO path (e.g., "projects/p1/docs/d1.pdf") |
| ocr_status | VARCHAR(50) | DEFAULT 'pending' | pending → processing → done/failed |
| chunk_count | INTEGER | DEFAULT 0 | Number of chunks created |
| metadata_json | JSONB | NULLABLE | Flexible metadata (parser output) |
| version | INTEGER | DEFAULT 1 | Increments on re-upload |
| created_at | TIMESTAMP | DEFAULT NOW() | Creation time |

**Model: Chunk**
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | VARCHAR(36) | PK | UUID primary key |
| document_id | VARCHAR(36) | FK → documents.id, CASCADE | Parent document |
| content | TEXT | NOT NULL | Chunk text (what gets embedded) |
| chunk_index | INTEGER | NOT NULL | Order within document (0, 1, 2, ...) |
| start_page | INTEGER | NULLABLE | Starting page number |
| end_page | INTEGER | NULLABLE | Ending page number |
| metadata_json | JSONB | NULLABLE | Section, headers, measurements |
| embedding_id | VARCHAR(255) | NULLABLE | Qdrant point ID |
| created_at | TIMESTAMP | DEFAULT NOW() | Creation time |

**Model: Entity**
| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | VARCHAR(36) | PK | UUID primary key |
| project_id | VARCHAR(36) | FK → projects.id, CASCADE | Parent project |
| name | VARCHAR(255) | NOT NULL | Entity name (e.g., "Bedroom 1") |
| entity_type | VARCHAR(100) | NOT NULL | building, floor, room, wall, beam, material, vendor |
| properties_json | JSONB | NULLABLE | Entity properties (dimensions, specs) |
| neo4j_id | VARCHAR(255) | NULLABLE | Neo4j node reference |
| created_at | TIMESTAMP | DEFAULT NOW() | Creation time |

**Relationships:**
```
Project 1──* Document 1──* Chunk
Project 1──* Entity
```

**Cascade deletes:**
- Deleting a Project deletes all its Documents, Chunks, and Entities
- Deleting a Document deletes all its Chunks

### civilmind/db/session.py

**Purpose:** FastAPI dependency that yields database sessions.

**What it does:**
1. Creates async session from factory
2. Yields session to endpoint
3. Commits on success
4. Rolls back on exception
5. Closes session always

**Why this pattern:**
- Automatic transaction management
- No need for manual commit/rollback in endpoints
- Session cleanup guaranteed (finally block)

## Commands

```bash
# Generate migration after model changes
alembic revision --autogenerate -m "description"

# Apply all pending migrations
alembic upgrade head

# Rollback one migration
alembic downgrade -1

# Check current version
alembic current

# Show migration history
alembic history
```

## Verification

```bash
# After starting PostgreSQL
alembic upgrade head

# Check tables exist

# Expected:

```
